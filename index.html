Scope作用域

​ Spring 管理的 bean 是根据 scope 来生成的，表示 bean 的作用域，共四种，默认是singleton.

  • Singleton：单例模式，表示通过 IoC 容器获取的 bean 是唯一的。
  • ProtoType：原型模式， 表示通过 IoC 容器获取的 bean 是不同的。
  • Request：请求， 表示在一次HTTP请求内有效。
  • session：会话，表示在一个用户会话中有效。

request 和 session 只适⽤于 Web 项⽬，⼤多数情况下，使⽤单例和原型较多。

prototype 模式当业务代码获取 IoC 容器中的 bean 时，Spring 才去调⽤⽆参构造创建对应的 bean。

singleton 模式⽆论业务代码是否获取 IoC 容器中的 bean，Spring 在加载 spring.xml 时就会创建 bean。

  • 好处是节省jvm中堆的资源，每次从ioc中获取对象只需要在栈中开辟新的空间存对象的引用即可
  • 坏处是无论是否获取对象，每次加载xml配置文件时，IoC都会创建一个bean.

Spring的继承

和java的继承不同，java的继承是类的层面，子类可以继承父类的内部结构和信息；Spring的继承是对象层面，子对象可以继承父对象的属性值。

  • 特性：Spring中的任意bean之间可以继承，前提是子对象中的属性续包含父对象中的属性

​ bean中继承的方式 parent='xxx'是将父对象中的所有property复制到子对象中的一个操作。

​ 所以即使父对象的scope是原型模式，依旧可以继承其属性值。

spring 的继承的关注点在于具体的对象，而不在于类。即不同的两个类的实例化对象可以完成继承（即使两个类不存在继承关系），前提是子对象包含所有父对象的成员变量。同时也可以在父对象的基础上添加其他属性

Spring的工厂方法

IoC是典型的工厂模式: IoC生产对象就像工厂生产产品一样，它会提前在工厂内生产好对象，我们只需要获取

AOP是代理模式

工厂方法

  • 工厂提供服务（方法），客户只需要满足服务的条件即可，解耦明显。每一环这需要关心自己的实现，不必关心另一环的实现。

IoC通过工厂模式创建bean的方式有两种

  • 静态工厂方法

    • 工厂的服务是静态的，工厂提供的方法是static方法。这些方法仅由工厂提供以及管理更新，跟客户或者调用端无关.

  • 实例工厂方法

    • 工厂提供的方法非static，客户端或调用端需要自己创建工厂对象然后再去调用工厂中的服务，所以管理工厂的相当于客户端了。与静态工厂方法相比，更灵活。

具体使用方式

Spring IoC的自动装载(Autowire)

IoC 负责创建对象，DI 负责完成对象的依赖注⼊，通过配置 property 标签的 ref 属性来完成，同时 Spring 提供了另外⼀种更加简便的依赖注⼊⽅式：⾃动装载，不需要⼿动配置 property，IoC 容器会⾃ 动选择 bean 完成注⼊。

⾃动装载有两种⽅式：

  • byName：通过属性名⾃动装载
  • byType：通过属性的数据类型⾃动装载
